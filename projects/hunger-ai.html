The Challenge

Food insecurity is a complex, multi-variable problem driven by politics, climate, economics, and logistics. The goal was to build a simulation engine that could model these interacting forces and generate realistic crisis scenarios — then use AI to propose actionable solutions.

The key technical challenge was prompt engineering: how do you instruct an LLM to behave as a geopolitical simulation engine that produces consistent, grounded outputs rather than generic responses? This required iterative refinement of system prompts and output schemas.

Skills Developed

Python OpenAI API Prompt Engineering Simulation Design Data Modeling

Design Process

Problem Research

Studied real-world food crisis data, supply chain disruption patterns, and UN hunger reports. Identified the key variables — crop yield, trade routes, political stability, climate events — that drive food insecurity.

Simulation Architecture

Designed a turn-based simulation engine in Python where each "turn" represents a time period. The LLM receives the current state, generates events and consequences, and the engine updates the world model accordingly.

Prompt Engineering & Iteration

Spent significant time refining system prompts to ensure the AI produced structured, realistic scenarios rather than generic text. Implemented output parsing with JSON schemas to make responses machine-readable.

Solution Generation

Built a secondary agent that analyzes crisis scenarios and proposes intervention strategies. Validated outputs against real-world case studies to assess feasibility and relevance.

What I Learned

LLMs are powerful but need guardrails. Without structured output schemas and careful system prompts, the AI would drift into generic advice or hallucinate scenarios. Learning to constrain creativity without killing it was the core skill here.

Domain expertise matters for AI applications. The simulation was only as good as my understanding of the problem domain. Building useful AI tools requires investing in the subject matter, not just the technology.

Iterative development is essential for AI. Unlike traditional software where you can test deterministically, AI outputs vary. This project taught me to think in terms of evaluation criteria and output quality distributions rather than binary pass/fail tests.
