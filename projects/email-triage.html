The Challenge

Email overload is a universal problem. The average professional spends hours daily sorting, reading, and responding to messages. The Email Triage Agent was built to automate the triage layer — intelligently classifying incoming email by urgency, topic, and required action, then drafting context-aware responses.

The technical challenge was building a reliable agent pipeline: connecting to email APIs, parsing varied email formats (HTML, plain text, thread chains), applying LLM classification with consistent results, and generating responses that match the user's writing style and intent.

Skills Developed

Design Process

Problem Analysis & Email Taxonomy

Analyzed my own email patterns to identify categories: urgent/action-required, informational, newsletters, transactional, and personal. Defined a classification schema with priority levels and expected response times for each category.

Email Ingestion Pipeline

Built an integration layer that connects to email providers via API, fetches new messages, strips HTML to extract meaningful content, and handles thread reconstruction. Implemented rate limiting and error handling for production reliability.

LLM Classification & Triage

Designed a multi-step prompt chain: first classify the email's category and urgency, then extract key action items, and finally generate a summary. Used structured output parsing to ensure consistent, machine-readable results from the LLM.

Response Drafting & UI

Built a Next.js dashboard that displays triaged emails with their classifications, summaries, and draft responses. Users can approve, edit, or reject drafts. The AI learns from edits to improve future response quality over time.

What I Learned

Agent architecture is about reliability, not intelligence. The flashy part is the LLM, but 80% of the work was building robust email parsing, error handling, and retry logic. An agent that's brilliant 90% of the time but crashes 10% is useless.

Prompt chains outperform single prompts. Breaking the classification task into sequential steps (categorize → extract actions → summarize → draft) produced dramatically better results than trying to do everything in one prompt. Each step is focused and verifiable.

User trust is earned incrementally. Nobody wants to hand their email to an AI on day one. Building confidence through transparent classification (showing why the AI made each decision) and keeping humans in the loop for sending responses was essential for adoption.
